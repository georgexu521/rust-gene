# Stuck - 卡住恢复

帮助你在遇到困难或卡住时恢复前进。

## 使用场景

当你遇到以下情况时使用此 Skill：
- 不知道下一步该做什么
- 代码无法编译
- 测试一直失败
- 想不出解决方案
- 感觉被问题困住

## 恢复策略

### 1. 重新审视问题
- 回到最初的需求
- 重新定义问题
- 检查假设是否正确

### 2. 分解问题
- 将大问题拆成小问题
- 先解决最简单的部分
- 逐步构建解决方案

### 3. 换个角度
- 尝试不同的方法
- 从结果倒推
- 寻找类似问题的解决方案

### 4. 寻求帮助
- 查阅文档
- 搜索解决方案
- 询问更有经验的人

## 具体步骤

### 步骤 1：暂停并呼吸
```markdown
1. 停止当前的挣扎
2. 深呼吸几次
3. 从问题中抽离
```

### 步骤 2：明确卡住的原因
```markdown
- 是知识不足？
- 是思路错误？
- 是工具问题？
- 是需求不清？
```

### 步骤 3：收集信息
```markdown
- 错误信息是什么？
- 最近做了什么改动？
- 什么情况下能正常工作？
- 有相关的文档或示例吗？
```

### 步骤 4：尝试不同方法
```markdown
- 方法 A：从头开始
- 方法 B：使用不同的库/工具
- 方法 C：简化需求
- 方法 D：分阶段实现
```

### 步骤 5：记录和学习
```markdown
- 记录遇到的问题
- 记录解决方案
- 记录学到的教训
```

## 常见卡住情况

### 编译错误卡住
```rust
// 策略：逐个修复
1. 先修复第一个错误
2. 重新编译
3. 重复直到没有错误
```

### 测试失败卡住
```rust
// 策略：隔离问题
1. 运行单个测试
2. 添加调试输出
3. 检查测试数据
4. 简化测试案例
```

### 设计卡住
```rust
// 策略：画图或写伪代码
1. 画出架构图
2. 写出伪代码
3. 找到核心问题
4. 实现最小可行版本
```

### 性能问题卡住
```rust
// 策略：测量和分析
1. 添加性能计数器
2. 使用 profiler
3. 找到热点
4. 优化关键路径
```

## 心理调适

### 承认卡住是正常的
- 每个程序员都会卡住
- 这是学习过程的一部分
- 不要因此感到沮丧

### 保持积极心态
- 相信问题可以解决
- 关注进展而非障碍
- 庆祝小胜利

### 管理期望
- 不要追求完美
- 接受迭代改进
- 先完成再完美

## 预防措施

### 避免卡住
- 定期提交代码
- 保持代码整洁
- 编写测试
- 文档化决策

### 建立支持系统
- 寻找导师
- 加入社区
- 建立知识库
- 分享经验

## 输出格式

恢复建议应该包含：
1. 问题诊断
2. 可能的原因
3. 建议的解决方案
4. 实施步骤
5. 预期结果
