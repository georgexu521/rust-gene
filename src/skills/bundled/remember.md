# Remember - 记忆管理

帮助你管理和利用对话历史和项目知识。

## 使用场景

当你需要处理以下任务时使用此 Skill：
- 记住重要的上下文信息
- 追踪项目状态和进度
- 管理代码库知识
- 跨会话保持连续性
- 提取和总结关键信息

## 记忆类型

### 1. 短期记忆 (会话内)
- 当前任务状态
- 最近的修改
- 临时变量和假设

### 2. 长期记忆 (跨会话)
- 项目架构
- 重要决策
- 代码规范
- 常见问题

### 3. 工作记忆
- 正在处理的任务
- 待办事项
- 阻塞问题

## 记忆系统

### MEMORY.md 结构
```markdown
# 项目记忆

## 架构决策
- 使用 SQLite 存储会话
- 采用 WebSocket 实时通信
- 使用 Rust 2021 edition

## 代码规范
- 错误处理使用 anyhow
- 日志使用 tracing
- 测试使用 #[tokio::test]

## 常见问题
- 编译错误：检查依赖版本
- 测试失败：清理 target 目录

## 待办事项
- [ ] 实现用户认证
- [ ] 添加单元测试
```

### USER.md 结构
```markdown
# 用户信息

## 偏好
- 使用中文交流
- 喜欢详细的解释
- 优先考虑性能

## 工作习惯
- 上午 9-12 点最高效
- 喜欢先规划后执行
- 重视代码质量

## 项目背景
- 正在学习 Rust
- 开发 AI 代理系统
- 目标是商业化
```

## 记忆操作

### 保存记忆
```rust
// 保存到 MEMORY.md
memory_save("架构使用 SQLite 存储会话数据")

// 保存到 USER.md
memory_save("用户喜欢中文交流", target="user")
```

### 加载记忆
```rust
// 加载相关记忆
let memories = memory_load("SQLite 会话")
```

### 搜索记忆
```rust
// 搜索关键词
memory_search("数据库")
```

## 知识提取

### 从对话中提取
- 关键决策
- 重要信息
- 用户偏好
- 项目状态

### 从代码中提取
- 架构模式
- 常用函数
- 配置选项
- 依赖关系

### 从文档中提取
- API 接口
- 使用说明
- 最佳实践

## 记忆管理策略

### 自动保存
- 重要决策自动保存
- 用户明确要求记住
- 跨会话关键信息

### 定期整理
- 合并重复信息
- 删除过时内容
- 重新组织结构

### 优先级排序
- 高优先级：用户偏好、项目架构
- 中优先级：代码规范、常见问题
- 低优先级：临时状态、调试信息

## 使用示例

### 场景 1：记住项目决策
```
用户：我们决定使用 SQLite 而不是 PostgreSQL
助手：我已记住这个重要决策。
[保存到 MEMORY.md]
```

### 场景 2：恢复上下文
```
用户：我们上次讨论到哪了？
助手：让我查看记忆...
[加载 MEMORY.md 和最近会话]
上次我们讨论了数据库选择，决定使用 SQLite。
```

### 场景 3：用户偏好
```
用户：请用中文解释
助手：好的，我会记住这个偏好。
[保存到 USER.md]
```

## 最佳实践

1. **及时保存**：重要信息立即保存
2. **分类存储**：不同类型信息分开存储
3. **定期清理**：删除过时和重复内容
4. **搜索友好**：使用关键词便于搜索
5. **版本控制**：跟踪记忆变更历史

## 输出格式

记忆操作结果应该包含：
1. 保存的信息摘要
2. 存储位置
3. 相关上下文
4. 搜索结果（如果有）
5. 建议的后续操作
