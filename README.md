# Priority Agent - 加权优先级桌面 Agent

## 项目愿景
解决 AI Agent 抓不住重点的问题，通过显式的权重系统让 AI 始终专注于最重要的事项。

## 核心理念

### 权重分层系统
```
项目目标
├── 一级任务 (权重和 = 100%)
│   ├── 任务 A: 40%
│   │   ├── 子任务 A1: 60% (占整体 24%)
│   │   └── 子任务 A2: 40% (占整体 16%)
│   ├── 任务 B: 35%
│   └── 任务 C: 25%
```

### AI 自动权重分析
- 分析代码结构
- 识别关键路径
- 考虑依赖关系
- 动态调整权重

## 技术栈
- **语言**: Rust
- **架构**: 参考 Claude Code，但加入权重系统
- **核心模块**:
  - `weight-engine`: 权重计算引擎
  - `task-analyzer`: 任务分析器
  - `priority-queue`: 优先级队列
  - `context-manager`: 上下文管理

## 与 Claude Code 的对比测试

### 测试场景设计
**复杂任务**: "实现一个用户认证系统"

**预期对比**:
| 维度 | Claude Code | Priority Agent |
|------|-------------|----------------|
| 执行顺序 | 线性，容易在细节上绕圈 | 按权重优先级，先核心后细节 |
| 重点把握 | 容易偏离 | 始终聚焦高权重任务 |
| 完成度感知 | 模糊 | 数学化进度计算 |
